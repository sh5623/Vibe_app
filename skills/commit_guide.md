# Skill: Project Commit Guide

이 스킬은 Vibe App 프로젝트의 변경 사항을 커밋할 때 따르는 표준 절차와 메시지 형식을 정의합니다.

## 🛠 작업 절차
1. `git branch --show-current` 명령어로 현재 브랜치를 확인합니다.
2. **[중요]** 만약 현재 브랜치가 `main`인 경우:
   - **직접 커밋을 중단**합니다.
   - 사용자에게 "현재 main 브랜치입니다. 새 브랜치를 생성하여 작업하시겠습니까?"라고 노티합니다.
3. 브랜치가 안전한 경우(feature/, hotfix/ 등), `git status`로 변경 사항을 확인합니다.
4. 관련 파일을 `git add` 합니다.
5. 아래 형식에 맞춰 `git commit`을 수행합니다.

## 🛡 브랜치 보호 규칙
- 이 프로젝트의 `main` 브랜치는 배포 전용입니다.
- AI는 절대 `main` 브랜치에 직접 커밋하거나 Push하지 않습니다.
- 모든 작업은 별도의 작업 브랜치에서 수행 후 PR을 통해 반영하는 것을 원칙으로 합니다.

## 📝 커밋 메시지 형식
- **Headline**: `type: 요약 내용 (핵심 키워드)`
  - 예: `fix: 모바일 반응형 디자인 개선 (베이지 배경)`
- **Body**: 작업 내용을 불렛 포인트로 상세히 기술
  - 예:
    - HeroName 폰트 크기 조정
    - 글로벌 배경색 복구 및 dev 페이지 격리
    - Stack 컴포넌트 데이터 업데이트

## ⚠️ 주의 사항
- 항상 한국어로 상세 내역을 작성합니다.
- UI 수정 시에는 어떤 환경(모바일/데스크탑)을 위한 것인지 명시합니다.
